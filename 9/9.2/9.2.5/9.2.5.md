# 9.2.5. Módulo 5 / Integrante 5

## Módulo de Monitoreo de Entrega

### Funcionalidad primaria del Módulo de Monitoreo de Entrega

#### Flujo elegido: Operación Completa de Monitoreo en Tiempo Real

- Descripción del flujo:
    - El operador registra una nueva operación de monitoreo, asigna contenedores y medios de transporte (vehículo o buque), activa el seguimiento GPS y sensores IoT, monitorea en tiempo real la ubicación y condiciones del contenedor, recibe notificaciones automáticas ante anomalías, registra incidencias si es necesario, y finalmente cierra la operación al completar la entrega al importador.

#### Justificación de la Elección

Este flujo fue seleccionado por las siguientes razones:

1. **Integración Multi-tabla:** Conecta directamente más de 10 tablas principales del esquema:
    > Operacion → OperacionMonitoreo → Contenedor → Sensor → Notificacion → Incidencia → PosicionContenedor → Vehiculo/Buque → Operador → Entrega

2. **Valor de Negocio:** Es la funcionalidad CORE del módulo. Sin este flujo, el sistema no cumple su propósito principal de trazabilidad logística.

3. **Complejidad Técnica:** Implica:
    - Transacciones complejas con múltiples inserciones relacionadas
    - Consultas geoespaciales para tracking GPS
    - Lógica de negocio para validaciones (umbrales de sensores, estados)
    - Agregaciones para reportes

#### Mapeo de Requisitos Funcionales

| **Requerimiento** | **Implementado en el Flujo** |
| -- | -- |
| RF01 | Registro de operación de monitoreo con asignación de contenedores, vehículos y operadores |
| RF02 | Captura automática de posiciones GPS (simulada con inserts periódicos) |
| RF03 | Lectura de datos de sensores IoT y almacenamiento histórico |
| RF04 | Generación de notificaciones cuando sensores superan umbrales |
| RF05 | Registro de incidencias asociadas a la operación |
| RF06 | Generación de reporte consolidado de la operación |
| RF07 | Visualización de posiciones en mapa (consulta de últimas posiciones) |
| RF08 | Registro de entrega final con validación |

#### Interfaces de Usuario Incluidas

Pantallas Principales Implementadas:

- **P01 - Registro de Operación de Monitoreo (CU01)**
    - Formulario para crear operación
    - Selección de contenedores disponibles
    - Asignación de vehículo/buque
    - Asignación automática de operador

- **P03 - Dashboard de Monitoreo en Tiempo Real (CU02, CU07)**
    - Visualización de operaciones activas
    - KPIs: contenedores en tránsito, alertas activas, operaciones del día
    - Mapa con posiciones actuales de contenedores

- **P05 - Vista Detallada de Contenedor (CU03)**
    - Información del contenedor
    - Lecturas actuales de sensores (temperatura, humedad, vibración)
    - Gráficos históricos de sensores

- **P07 - Panel de Notificaciones (CU04)**
    - Lista de notificaciones generadas
    - Filtros por tipo (Alerta, Advertencia, Crítica)
    - Notificaciones no leídas destacadas

- **P08 - Registro de Incidencias (CU05)**
    - Formulario para reportar incidencias
    - Vinculación con operación de monitoreo
    - Clasificación por tipo y severidad

- **P10 - Entrega Final (CU08)**
    - Registro de entrega al importador
    - Adjuntar evidencia digital
    - Cierre de operación

#### Consultas y Reportes Implementados

##### **Consultas Principales:**

1. **Búsqueda de Operaciones por Estado:**

```SQL
-- Muestra operaciones con su estado y cuenta de contenedores asociados
    SELECT 
        o.codigo AS codigo_operacion,
        o.fecha_inicio,
        o.fecha_fin,
        eo.nombre AS estado,
        COUNT(DISTINCT cv.id_contenedor) AS contenedores_vehiculo,
        COUNT(DISTINCT cb.id_contenedor) AS contenedores_buque,
        COUNT(DISTINCT cv.id_contenedor) + COUNT(DISTINCT cb.id_contenedor) AS total_contenedores
    FROM monitoreo.Operacion o
    JOIN monitoreo.EstadoOperacion eo ON o.id_estado_operacion = eo.id_estado_operacion
    JOIN monitoreo.OperacionMonitoreo om ON o.id_operacion = om.id_operacion
    LEFT JOIN monitoreo.OperacionMonitoreoVehiculo omv ON om.id_operacion_monitoreo = omv.id_operacion_monitoreo
    LEFT JOIN monitoreo.ContenedorVehiculo cv ON omv.id_vehiculo = cv.id_vehiculo 
        AND cv.fecha_asignacion <= CURRENT_DATE
    LEFT JOIN monitoreo.OperacionMonitoreoBuque omb ON om.id_operacion_monitoreo = omb.id_operacion_monitoreo
    LEFT JOIN monitoreo.ContenedorBuque cb ON omb.id_buque = cb.id_buque 
        AND cb.fecha_asignacion <= CURRENT_DATE
    WHERE eo.nombre = 'En curso' -- Cambiar según el estado deseado
    GROUP BY o.id_operacion, o.codigo, o.fecha_inicio, o.fecha_fin, eo.nombre
    ORDER BY o.fecha_inicio DESC;
```

2. **Tracking GPS - Últimas Posiciones:**

```sql
-- Muestra el historial de posiciones de contenedores con posición anterior
    SELECT 
        c.codigo AS codigo_contenedor,
        pc.latitud,
        pc.longitud,
        pc.fecha_hora,
        LAG(pc.latitud) OVER (PARTITION BY c.id_contenedor ORDER BY pc.fecha_hora) AS lat_anterior,
        LAG(pc.longitud) OVER (PARTITION BY c.id_contenedor ORDER BY pc.fecha_hora) AS lon_anterior,
        LAG(pc.fecha_hora) OVER (PARTITION BY c.id_contenedor ORDER BY pc.fecha_hora) AS fecha_anterior
    FROM monitoreo.Contenedor c
    JOIN monitoreo.PosicionContenedor pc ON c.id_contenedor = pc.id_contenedor
    WHERE c.id_contenedor IN (
        -- Contenedores de una operación específica
        SELECT DISTINCT cv.id_contenedor 
        FROM monitoreo.OperacionMonitoreoVehiculo omv
        JOIN monitoreo.ContenedorVehiculo cv ON omv.id_vehiculo = cv.id_vehiculo
        WHERE omv.id_operacion_monitoreo = 'UUID_AQUI' -- Reemplazar con UUID real
        UNION
        SELECT DISTINCT cb.id_contenedor 
        FROM monitoreo.OperacionMonitoreoBuque omb
        JOIN monitoreo.ContenedorBuque cb ON omb.id_buque = cb.id_buque
        WHERE omb.id_operacion_monitoreo = 'UUID_AQUI' -- Reemplazar con UUID real
    )
    ORDER BY c.codigo, pc.fecha_hora DESC;
```

3. **Posición actual de todos los contenedores:**

```sql
-- Para visualización en mapa en tiempo real
    WITH UltimaPosicion AS (
        SELECT 
            id_contenedor,
            latitud,
            longitud,
            fecha_hora,
            ROW_NUMBER() OVER (PARTITION BY id_contenedor ORDER BY fecha_hora DESC) AS rn
        FROM monitoreo.PosicionContenedor
    )
    SELECT 
        c.codigo AS codigo_contenedor,
        c.peso,
        c.capacidad,
        ec.nombre AS estado_contenedor,
        up.latitud AS lat_actual,
        up.longitud AS lon_actual,
        up.fecha_hora AS ultima_actualizacion
    FROM monitoreo.Contenedor c
    LEFT JOIN UltimaPosicion up ON c.id_contenedor = up.id_contenedor AND up.rn = 1
    JOIN monitoreo.EstadoContenedor ec ON c.id_estado_contenedor = ec.id_estado_contenedor
    WHERE ec.nombre = 'Disponible'
    ORDER BY c.codigo;
```

4. **Alertas críticas activas:**

```sql
-- Muestra notificaciones generadas por sensores con información del contenedor
    SELECT 
        n.codigo AS codigo_notificacion,
        c.codigo AS codigo_contenedor,
        s.codigo AS codigo_sensor,
        s.nombre AS nombre_sensor,
        ts.nombre AS tipo_sensor,
        tn.nombre AS tipo_notificacion,
        n.valor AS valor_sensor,
        n.fecha_hora,
        EXTRACT(EPOCH FROM (CURRENT_TIMESTAMP - n.fecha_hora))/60 AS minutos_desde_alerta
    FROM monitoreo.Notificacion n
    JOIN monitoreo.Sensor s ON n.id_sensor = s.id_sensor
    JOIN monitoreo.TipoSensor ts ON s.id_tipo_sensor = ts.id_tipo_sensor
    JOIN monitoreo.Contenedor c ON s.id_contenedor = c.id_contenedor
    JOIN monitoreo.TipoNotificacion tn ON n.id_tipo_notificacion = tn.id_tipo_notificacion
    WHERE tn.nombre IN ('Crítica', 'Advertencia') -- Filtrar por tipo
    AND n.fecha_hora >= CURRENT_DATE - INTERVAL '7 days' -- Últimos 7 días
    ORDER BY n.fecha_hora DESC;
```

5. **Historial de sensonres por contenedor:**

```sql
-- Muestra todas las notificaciones (valores de sensores) de un contenedor específico
    SELECT 
        c.codigo AS codigo_contenedor,
        s.codigo AS codigo_sensor,
        ts.nombre AS tipo_sensor,
        n.valor,
        tn.nombre AS tipo_notificacion,
        n.fecha_hora,
        DATE_TRUNC('hour', n.fecha_hora) AS hora_grupo
    FROM monitoreo.Contenedor c
    JOIN monitoreo.Sensor s ON c.id_contenedor = s.id_contenedor
    JOIN monitoreo.TipoSensor ts ON s.id_tipo_sensor = ts.id_tipo_sensor
    JOIN monitoreo.Notificacion n ON s.id_sensor = n.id_sensor
    JOIN monitoreo.TipoNotificacion tn ON n.id_tipo_notificacion = tn.id_tipo_notificacion
    WHERE c.codigo = 'CONT-001' -- Cambiar por código real
    ORDER BY n.fecha_hora DESC;
```

##### **Reportes Consolidados (CU06):**

1. **Reporte de operación completo:**

```sql
-- Consolida toda la información de una operación de monitoreo
    SELECT 
        r.codigo AS codigo_reporte,
        r.fecha_reporte,
        o.codigo AS codigo_operacion,
        o.fecha_inicio,
        o.fecha_fin,
        eo.nombre AS estado_operacion,
        
        -- Contenedores monitoreados
        (SELECT COUNT(DISTINCT cv.id_contenedor) 
        FROM monitoreo.OperacionMonitoreoVehiculo omv
        JOIN monitoreo.ContenedorVehiculo cv ON omv.id_vehiculo = cv.id_vehiculo
        WHERE omv.id_operacion_monitoreo = om.id_operacion_monitoreo) +
        (SELECT COUNT(DISTINCT cb.id_contenedor) 
        FROM monitoreo.OperacionMonitoreoBuque omb
        JOIN monitoreo.ContenedorBuque cb ON omb.id_buque = cb.id_buque
        WHERE omb.id_operacion_monitoreo = om.id_operacion_monitoreo) AS total_contenedores,
        
        -- Notificaciones generadas
        (SELECT COUNT(*) 
        FROM monitoreo.Notificacion n2 
        WHERE n2.id_reporte = r.id_reporte) AS total_notificaciones,
        
        -- Incidencias registradas
        (SELECT COUNT(*) 
        FROM monitoreo.IncidenciaReporte ir
        WHERE ir.id_reporte = r.id_reporte) AS total_incidencias,
        
        -- Operadores asignados
        (SELECT COUNT(DISTINCT oom.id_operador)
        FROM monitoreo.OperadorOperacionMonitoreo oom
        WHERE oom.id_operacion_monitoreo = om.id_operacion_monitoreo) AS operadores_asignados

    FROM monitoreo.Reporte r
    JOIN monitoreo.Notificacion n ON r.id_reporte = n.id_reporte
    JOIN monitoreo.Sensor s ON n.id_sensor = s.id_sensor
    JOIN monitoreo.Contenedor c ON s.id_contenedor = c.id_contenedor
    JOIN monitoreo.ContenedorVehiculo cv ON c.id_contenedor = cv.id_contenedor
    JOIN monitoreo.OperacionMonitoreoVehiculo omv ON cv.id_vehiculo = omv.id_vehiculo
    JOIN monitoreo.OperacionMonitoreo om ON omv.id_operacion_monitoreo = om.id_operacion_monitoreo
    JOIN monitoreo.Operacion o ON om.id_operacion = o.id_operacion
    JOIN monitoreo.EstadoOperacion eo ON o.id_estado_operacion = eo.id_estado_operacion
    WHERE r.id_reporte = 'UUID_AQUI' -- Reemplazar con UUID real del reporte
    GROUP BY r.codigo, r.fecha_reporte, o.codigo, o.fecha_inicio, o.fecha_fin, 
            eo.nombre, om.id_operacion_monitoreo, r.id_reporte;
```

2. **Top 10 contenedores con más alertas:**
```sql
-- Reporte para administradores mostrando contenedores problemáticos
    SELECT 
        c.codigo AS codigo_contenedor,
        tc.nombre AS tipo_contenedor,
        ec.nombre AS estado,
        COUNT(n.id_notificacion) AS total_alertas,
        COUNT(CASE WHEN tn.nombre = 'Crítica' THEN 1 END) AS alertas_criticas,
        COUNT(CASE WHEN tn.nombre = 'Advertencia' THEN 1 END) AS alertas_advertencia,
        MAX(n.fecha_hora) AS ultima_alerta,
        EXTRACT(DAY FROM (CURRENT_TIMESTAMP - MAX(n.fecha_hora))) AS dias_desde_ultima_alerta
    FROM monitoreo.Contenedor c
    JOIN monitoreo.TipoContenedor tc ON c.id_tipo_contenedor = tc.id_tipo_contenedor
    JOIN monitoreo.EstadoContenedor ec ON c.id_estado_contenedor = ec.id_estado_contenedor
    LEFT JOIN monitoreo.Sensor s ON c.id_contenedor = s.id_contenedor
    LEFT JOIN monitoreo.Notificacion n ON s.id_sensor = n.id_sensor
    LEFT JOIN monitoreo.TipoNotificacion tn ON n.id_tipo_notificacion = tn.id_tipo_notificacion
    WHERE n.fecha_hora >= CURRENT_DATE - INTERVAL '30 days' -- Últimos 30 días
    GROUP BY c.id_contenedor, c.codigo, tc.nombre, ec.nombre
    HAVING COUNT(n.id_notificacion) > 0
    ORDER BY total_alertas DESC, alertas_criticas DESC
    LIMIT 10;
```

3. **Operadores con mayor carga de operaciones:**

```sql
-- Reporte de rendimiento de operadores
SELECT 
    e.nombre || ' ' || e.apellido AS operador,
    e.codigo AS codigo_empleado,
    re.nombre AS rol,
    op.zona_monitoreo,
    op.turno,
    COUNT(DISTINCT oom.id_operacion_monitoreo) AS operaciones_asignadas,
    COUNT(DISTINCT i.id_incidencia) AS incidencias_reportadas,
FROM monitoreo.Operador op
JOIN monitoreo.Empleado e ON op.id_empleado = e.id_empleado
JOIN monitoreo.EspecialidadEmpleado re ON e.id_especialidad_empleado = re.id_especialidad_empleado
LEFT JOIN monitoreo.OperadorOperacionMonitoreo oom ON op.id_operador = oom.id_operador
LEFT JOIN monitoreo.OperacionMonitoreo om ON oom.id_operacion_monitoreo = om.id_operacion_monitoreo
LEFT JOIN monitoreo.Usuario u ON e.id_empleado = u.id_empleado
LEFT JOIN monitoreo.Incidencia i ON u.id_usuario = i.id_usuario
WHERE oom.fecha_realizacion >= CURRENT_DATE - INTERVAL '30 days' -- Últimos 30 días
GROUP BY op.id_operador, e.nombre, e.apellido, e.codigo, re.nombre, 
         op.zona_monitoreo, op.turno
ORDER BY operaciones_asignadas DESC;
```

4. **Resumen de incidencias por operación:**

```sql
-- Detalle de incidencias registradas en operaciones de monitoreo
SELECT 
    o.codigo AS codigo_operacion,
    i.codigo AS codigo_incidencia,
    ti.nombre AS tipo_incidencia,
    i.grado_severidad,
    i.descripcion,
    ei.nombre AS estado_incidencia,
    i.fecha_hora AS fecha_registro,
    e.nombre || ' ' || e.apellido AS reportado_por,
    COUNT(ir.id_incidencia_reporte) AS veces_en_reportes
FROM monitoreo.Incidencia i
JOIN monitoreo.TipoIncidencia ti ON i.id_tipo_incidencia = ti.id_tipo_incidencia
JOIN monitoreo.EstadoIncidencia ei ON i.id_estado_incidencia = ei.id_estado_incidencia
JOIN monitoreo.Operacion o ON i.id_operacion = o.id_operacion
JOIN monitoreo.Usuario u ON i.id_usuario = u.id_usuario
JOIN monitoreo.Empleado e ON u.id_empleado = e.id_empleado
LEFT JOIN monitoreo.IncidenciaReporte ir ON i.id_incidencia = ir.id_incidencia
WHERE o.fecha_inicio >= CURRENT_DATE - INTERVAL '30 days' -- Últimas operaciones
GROUP BY o.codigo, i.codigo, ti.nombre, i.grado_severidad, i.descripcion, 
         ei.nombre, i.fecha_hora, e.nombre, e.apellido
ORDER BY i.grado_severidad DESC, i.fecha_hora DESC;
```

5. **Entregas completadas por importador:**

```sql
-- Reporte de entregas realizadas
SELECT 
    imp.codigo AS codigo_importador,
    imp.razon_social,
    COUNT(e.id_entrega) AS total_entregas,
    COUNT(CASE WHEN ee.nombre = 'Completada' THEN 1 END) AS entregas_completadas,
    COUNT(CASE WHEN ee.nombre = 'Pendiente' THEN 1 END) AS entregas_pendientes,
    MAX(e.fecha_entrega) AS ultima_entrega,
    STRING_AGG(DISTINCT c.codigo, ', ') AS contenedores_entregados
FROM monitoreo.Importador imp
LEFT JOIN monitoreo.Entrega e ON imp.id_importador = e.id_importador
LEFT JOIN monitoreo.EstadoEntrega ee ON e.id_estado_entrega = ee.id_estado_entrega
LEFT JOIN monitoreo.Contenedor c ON e.id_contenedor = c.id_contenedor
WHERE e.fecha_entrega >= CURRENT_DATE - INTERVAL '90 days' -- Últimos 3 meses
GROUP BY imp.id_importador, imp.codigo, imp.razon_social
ORDER BY total_entregas DESC;
```

6. **Tracking completo de vehículos en operaciones activas:**

```sql
-- Muestra la posición actual de vehículos asignados a operaciones
WITH UltimaPosicionVehiculo AS (
    SELECT 
        id_vehiculo,
        latitud,
        longitud,
        fecha_hora,
        ROW_NUMBER() OVER (PARTITION BY id_vehiculo ORDER BY fecha_hora DESC) AS rn
    FROM monitoreo.PosicionVehiculo
)
SELECT 
    o.codigo AS codigo_operacion,
    v.placa,
    tv.nombre AS tipo_vehiculo,
    ev.nombre AS estado_vehiculo,
    upv.latitud AS lat_actual,
    upv.longitud AS lon_actual,
    upv.fecha_hora AS ultima_actualizacion,
    COUNT(DISTINCT cv.id_contenedor) AS contenedores_transportados
FROM monitoreo.Operacion o
JOIN monitoreo.OperacionMonitoreo om ON o.id_operacion = om.id_operacion
JOIN monitoreo.OperacionMonitoreoVehiculo omv ON om.id_operacion_monitoreo = omv.id_operacion_monitoreo
JOIN monitoreo.Vehiculo v ON omv.id_vehiculo = v.id_vehiculo
JOIN monitoreo.TipoVehiculo tv ON v.id_tipo_vehiculo = tv.id_tipo_vehiculo
JOIN monitoreo.EstadoVehiculo ev ON v.id_estado_vehiculo = ev.id_estado_vehiculo
LEFT JOIN UltimaPosicionVehiculo upv ON v.id_vehiculo = upv.id_vehiculo AND upv.rn = 1
LEFT JOIN monitoreo.ContenedorVehiculo cv ON v.id_vehiculo = cv.id_vehiculo
WHERE o.id_estado_operacion = (SELECT id_estado_operacion FROM monitoreo.EstadoOperacion WHERE nombre = 'En curso')
GROUP BY o.codigo, v.placa, tv.nombre, ev.nombre, upv.latitud, upv.longitud, upv.fecha_hora
ORDER BY o.codigo;
```

7. **Sensores activos por contenedor con última lectura**

```sql
-- Para el dashboard de monitoreo de sensores
    SELECT 
        c.codigo AS codigo_contenedor,
        s.codigo AS codigo_sensor,
        s.nombre AS nombre_sensor,
        ts.nombre AS tipo_sensor,
        rs.nombre AS rol_sensor,
        n.valor AS ultima_lectura,
        tn.nombre AS tipo_notificacion,
        n.fecha_hora AS fecha_ultima_lectura,
		ec.nombre AS disponibilidad,
        EXTRACT(HOUR FROM (CURRENT_TIMESTAMP - n.fecha_hora)) AS horas_sin_actualizar
    FROM monitoreo.Contenedor c
    JOIN monitoreo.Sensor s ON c.id_contenedor = s.id_contenedor
    JOIN monitoreo.TipoSensor ts ON s.id_tipo_sensor = ts.id_tipo_sensor
    JOIN monitoreo.RolSensor rs ON s.id_rol_sensor = rs.id_rol_sensor
    LEFT JOIN LATERAL (
        SELECT n2.valor, n2.fecha_hora, n2.id_tipo_notificacion
        FROM monitoreo.Notificacion n2
        WHERE n2.id_sensor = s.id_sensor
        ORDER BY n2.fecha_hora DESC
        LIMIT 1
    ) n ON true
    LEFT JOIN monitoreo.TipoNotificacion tn ON n.id_tipo_notificacion = tn.id_tipo_notificacion
    LEFT JOIN monitoreo.EstadoContenedor ec ON c.id_estado_contenedor = ec.id_estado_contenedor
    WHERE ec.nombre = 'Disponible'
    ORDER BY c.codigo, s.codigo;
```
