# 9.2.1. Módulo 1 / Integrante 1

## Módulo de Gestión de Reservas

### Funcionalidad primaria del Módulo de Gestión de Reservas

#### Flujo elegido: Proceso Completo de Reserva de Transporte Marítimo

- Descripción del flujo:
    - El flujo inicia cuando un agente de reservas recibe la solicitud de un cliente para transportar contenedores. Primero, se valida la existencia del cliente en el sistema o se registra si es nuevo. Luego, el agente busca y selecciona una ruta marítima disponible que cumpla con los requisitos de origen y destino del cliente. Una vez confirmada la ruta, se procede a seleccionar un buque disponible que tenga capacidad suficiente y certificaciones vigentes. Posteriormente, se asignan los contenedores disponibles según el tipo y cantidad solicitada por el cliente, verificando su estado y compatibilidad. Con todos los elementos configurados, se genera una vista preliminar de la reserva donde se consolida la información: cliente, ruta, buque, contenedores y cálculo del pago total basado en las tarifas de ruta y costos de contenedores. El agente revisa y confirma la información; si requiere ajustes, puede modificar la selección de contenedores o cambiar el buque. Finalmente, al confirmar, se registra la reserva en estado "Confirmada" y se vinculan las operaciones terrestres y marítimas necesarias para ejecutar el transporte. El sistema actualiza automáticamente el estado de los contenedores a "En tránsito" y genera el código único de reserva para seguimiento. Todo este proceso se realiza principalmente a través de la pantalla P01 (Registro de Reservas), que integra todas las funcionalidades necesarias para completar el proceso de reserva.

#### Justificación de la Elección

La elección del flujo de Proceso Completo de Reserva de Transporte Marítimo como funcionalidad primaria se fundamenta en:

1. **Integración Multi-tabla Compleja:** Este flujo conecta directamente más de 12 tablas del esquema de base de datos:
    > Cliente → AgenteReservas → Reserva → Buque → Ruta → Contenedor → TipoContenedor → ReservaContenedor → ReservaOperacionMaritima → ReservaOperacionTerrestre → EstadoReserva → EstadoEmbarcacion → EstadoContenedor

2. **Núcleo del Negocio:** Es la funcionalidad CORE del sistema de gestión logística. Sin este flujo, el módulo no cumple su propósito principal de coordinar el transporte de mercancías. Representa el valor agregado real del negocio.

3. **Complejidad Transaccional:** Involucra múltiples validaciones de negocio
    - Validación de disponibilidad de buques por capacidad y estado
    - Verificación de certificaciones vigentes de embarcaciones
    - Validación de disponibilidad de contenedores por tipo y estado
    - Cálculo automático de costos (tarifa de ruta + costo de contenedores)
    - Actualización coordinada de estados en múltiples entidades

4. **Proceso No-CRUD:** Cumple con la restricción de no enfocarse en operaciones CRUD simples. Se trata de un proceso de negocio completo que
    - Requiere múltiples consultas de disponibilidad
    - Ejecuta validaciones de reglas de negocio
    - Realiza inserciones coordinadas en múltiples tablas relacionadas
    - Actualiza estados de manera transaccional

#### Interfaces de Usuario Incluidas

Pantallas Principales Implementadas:

- **P01 - Registro de Reservas (CU01)**
    - Validar o registrar cliente en el sistema
    - Seleccionar tipo de servicio y datos de carga
    - Buscar y seleccionar ruta marítima disponible
    - Seleccionar buque con capacidad suficiente
    - Asignar contenedores disponibles según tipo y cantidad
    - Generar vista preliminar con cálculo de pago total
    - Confirmar reserva y vincular operaciones marítimas/terrestres
    - Actualizar estado de contenedores a "En tránsito"

- **P02 - Listado de Reservas (CU02)**
    - Buscar reservas por ID, código o rango de fechas
    - Filtrar por estado (Confirmada, Activa, Finalizada, Cancelada)
    - Visualizar información resumida de cada reserva
    - Acceder a detalles completos de la reserva
    - Actualizar o modificar reserva existente

- **P03 - Gestión de Clientes e Historial (CU03)**
    - Consultar perfil completo del cliente
    - Visualizar historial de servicios y reservas anteriores
    - Actualizar información de contacto (teléfonos, email, dirección)
    - Ver estadísticas de cliente (total reservas, ingresos generados)
    - Mostrar rutas utilizadas y última atención recibida

- **P04 - Administración de Tarifas (CU04)**
    - Consultar tarifas por ruta y tipo de contenedor
    - Generar cotizaciones automáticas basadas en parámetros
    - Aplicar descuentos según volumen (5% por 5+, 10% por 10+)
    - Calcular total con descuentos incluidos
    - Mostrar validez de cotización (15 días)

- **P05 - Dashboard de Reservas (CU05)**
    - Visualizar KPIs: reservas activas, ocupación promedio de buques
    - Mostrar ingresos proyectados y contenedores en tránsito
    - Indicar cantidad de buques disponibles
    - Mostrar clientes activos del mes
    - Reportes de ocupación por ruta y programación

**1. Panel de Reservas Activas**
```sql
-- Muestra reservas activas con detalle de cliente, buque, ruta y costos
SELECT 
    r.codigo AS codigo_reserva,
    c.razon_social AS cliente,
    c.ruc,
    er.nombre AS estado_reserva,
    r.fecha_registro,
    r.pago_total,
    b.nombre AS nombre_buque,
    b.matricula,
    rt.codigo AS codigo_ruta,
    rt.origen,
    rt.destino,
    rt.duracion AS duracion_dias,
    rt.tarifa AS tarifa_ruta,
    COUNT(DISTINCT rc.id_contenedor) AS cantidad_contenedores,
    SUM(rc.cantidad) AS total_unidades_contenedor,
    STRING_AGG(DISTINCT tc.nombre, ', ') AS tipos_contenedor
FROM gestion_reserva.Reserva r
INNER JOIN gestion_reserva.Cliente c ON r.ruc_cliente = c.ruc
INNER JOIN gestion_reserva.EstadoReserva er ON r.id_estado_reserva = er.id_estado_reserva
INNER JOIN gestion_reserva.Buque b ON r.matricula_buque = b.matricula
INNER JOIN gestion_reserva.Ruta rt ON r.id_ruta = rt.id_ruta
LEFT JOIN gestion_reserva.ReservaContenedor rc ON r.id_reserva = rc.id_reserva
LEFT JOIN gestion_reserva.Contenedor cnt ON rc.id_contenedor = cnt.id_contenedor
LEFT JOIN gestion_reserva.TipoContenedor tc ON cnt.id_tipo_contenedor = tc.id_tipo_contenedor
WHERE er.nombre IN ('Confirmada', 'Activa')
GROUP BY r.id_reserva, r.codigo, c.razon_social, c.ruc, er.nombre, 
         r.fecha_registro, r.pago_total, b.nombre, b.matricula, 
         rt.codigo, rt.origen, rt.destino, rt.duracion, rt.tarifa
```

**2. Búsqueda de Rutas Disponibles**
```sql

SELECT 
    rt.codigo AS codigo_ruta,

    rt.destino,
    rt.duracion AS duracion_dias,
    rt.tarifa AS tarifa_base,
    COUNT(DISTINCT r.id_reserva) AS reservas_totales,
    COUNT(DISTINCT CASE 
        WHEN er.nombre IN ('Confirmada', 'Activa') 
        THEN r.id_reserva 
    END) AS reservas_activas,
    MAX(r.fecha_registro) AS ultima_reserva

LEFT JOIN gestion_reserva.Reserva r ON rt.id_ruta = r.id_ruta

HAVING COUNT(DISTINCT CASE WHEN er.nombre IN ('Confirmada', 'Activa') THEN r.id_reserva END) < 10  -- Capacidad máxima

```

**3. Buques Disponibles con Capacidad**
```sql
-- Lista buques con certificaciones vigentes y capacidad disponible
SELECT 
    b.matricula,
    b.nombre AS nombre_buque,
    b.capacidad AS capacidad_total,
    b.peso AS peso_maximo_toneladas,
    b.ubicacion_actual,
    ee.nombre AS estado_embarcacion,
    COUNT(DISTINCT cb.id_certificacion) AS certificaciones_vigentes,
    STRING_AGG(DISTINCT cert.nombre, ', ') AS nombres_certificaciones,
    COALESCE(
        (
            SELECT SUM(rc.cantidad)
            FROM gestion_reserva.Reserva r_uso
            INNER JOIN gestion_reserva.ReservaContenedor rc ON r_uso.id_reserva = rc.id_reserva
            INNER JOIN gestion_reserva.EstadoReserva er_uso ON r_uso.id_estado_reserva = er_uso.id_estado_reserva
            WHERE r_uso.matricula_buque = b.matricula
            AND er_uso.nombre IN ('Confirmada', 'Activa')
        ), 0
    ) AS contenedores_reservados,
    (
        b.capacidad - COALESCE(
            (
                SELECT SUM(rc.cantidad)
                FROM gestion_reserva.Reserva r_uso
                INNER JOIN gestion_reserva.ReservaContenedor rc ON r_uso.id_reserva = rc.id_reserva
                INNER JOIN gestion_reserva.EstadoReserva er_uso ON r_uso.id_estado_reserva = er_uso.id_estado_reserva
                WHERE r_uso.matricula_buque = b.matricula
                AND er_uso.nombre IN ('Confirmada', 'Activa')
            ), 0
        )
    ) AS espacios_disponibles
FROM gestion_reserva.Buque b
INNER JOIN gestion_reserva.EstadoEmbarcacion ee ON b.id_estado_embarcacion = ee.id_estado_embarcacion
LEFT JOIN gestion_reserva.CertificacionBuque cb ON b.id_buque = cb.id_buque 
    AND cb.fecha_vencimiento >= CURRENT_DATE
LEFT JOIN gestion_reserva.Certificacion cert ON cb.id_certificacion = cert.id_certificacion
WHERE ee.nombre IN ('Disponible', 'En operación')
GROUP BY b.id_buque, b.matricula, b.nombre, b.capacidad, b.peso, 
         b.ubicacion_actual, ee.nombre
HAVING (
    b.capacidad - COALESCE(
        (
            SELECT SUM(rc.cantidad)
            FROM gestion_reserva.Reserva r_uso
            INNER JOIN gestion_reserva.ReservaContenedor rc ON r_uso.id_reserva = rc.id_reserva
            INNER JOIN gestion_reserva.EstadoReserva er_uso ON r_uso.id_estado_reserva = er_uso.id_estado_reserva
            WHERE r_uso.matricula_buque = b.matricula
            AND er_uso.nombre IN ('Confirmada', 'Activa')
        ), 0
    )
) > 0  -- Solo buques con espacio disponible
ORDER BY espacios_disponibles DESC, certificaciones_vigentes DESC;
```

**6. Generación de Cotización Automática**
```sql
-- Genera cotización basada en ruta y tipo de contenedor
SELECT 
    'COT-' || TO_CHAR(CURRENT_DATE, 'YYYYMMDD') || '-' || LPAD(nextval('seq_cotizacion')::TEXT, 4, '0') AS codigo_cotizacion,
    c.razon_social AS cliente,
    c.ruc,
    rt.codigo AS codigo_ruta,
    rt.origen,
    rt.destino,
    rt.duracion AS duracion_dias,
    rt.tarifa AS tarifa_base_ruta,
    tc.nombre AS tipo_contenedor,
    tc.codigo AS codigo_tipo_contenedor,
    5 AS cantidad_contenedores,  -- Parámetro de entrada
    tc.costo AS costo_unitario_contenedor,
    (tc.costo * 5) AS subtotal_contenedores,
    rt.tarifa AS subtotal_ruta,
    ((tc.costo * 5) + rt.tarifa) AS total_sin_descuento,
    CASE 
        WHEN 5 >= 10 THEN 0.10  -- 10% descuento por 10+ contenedores
        WHEN 5 >= 5 THEN 0.05   -- 5% descuento por 5+ contenedores
        ELSE 0.00
    END AS porcentaje_descuento,
    ((tc.costo * 5) + rt.tarifa) * (1 - CASE 
        WHEN 5 >= 10 THEN 0.10
        WHEN 5 >= 5 THEN 0.05
        ELSE 0.00
    END) AS total_con_descuento,
    CURRENT_DATE AS fecha_cotizacion,
    CURRENT_DATE + INTERVAL '15 days' AS validez_cotizacion,
    'Cotización sujeta a disponibilidad. Precios en USD.' AS observaciones
FROM gestion_reserva.Cliente c
CROSS JOIN gestion_reserva.Ruta rt
CROSS JOIN gestion_reserva.TipoContenedor tc
WHERE c.ruc = '20481234567'  -- RUC del cliente
AND rt.codigo = 'RUT-001'     -- Código de ruta seleccionada
AND tc.codigo = 'T-001';      -- Tipo de contenedor seleccionado
```

**7. Dashboard - Indicadores Clave de Rendimiento (KPIs)**
```sql
-- KPIs principales del módulo de reservas
WITH ReservasActivas AS (
    SELECT COUNT(*) AS total
    FROM gestion_reserva.Reserva r
    INNER JOIN gestion_reserva.EstadoReserva er ON r.id_estado_reserva = er.id_estado_reserva
    WHERE er.nombre IN ('Confirmada', 'Activa')
),
OcupacionBuques AS (
    SELECT 
        AVG(
            (COALESCE(contenedores_uso, 0)::NUMERIC / NULLIF(b.capacidad, 0)) * 100
        ) AS promedio_ocupacion
    FROM gestion_reserva.Buque b
    LEFT JOIN (
        SELECT 
            r.matricula_buque,
            SUM(rc.cantidad) AS contenedores_uso
        FROM gestion_reserva.Reserva r
        INNER JOIN gestion_reserva.ReservaContenedor rc ON r.id_reserva = rc.id_reserva
        INNER JOIN gestion_reserva.EstadoReserva er ON r.id_estado_reserva = er.id_estado_reserva
        WHERE er.nombre IN ('Confirmada', 'Activa')
        GROUP BY r.matricula_buque
    ) uso ON b.matricula = uso.matricula_buque
),
IngresosProyectados AS (
    SELECT COALESCE(SUM(r.pago_total), 0) AS total
    FROM gestion_reserva.Reserva r
    INNER JOIN gestion_reserva.EstadoReserva er ON r.id_estado_reserva = er.id_estado_reserva
    WHERE er.nombre IN ('Confirmada', 'Activa')
),
ContenedoresTransito AS (
    SELECT COUNT(DISTINCT c.id_contenedor) AS total
    FROM gestion_reserva.Contenedor c
    INNER JOIN gestion_reserva.EstadoContenedor ec ON c.id_estado_contenedor = ec.id_estado_contenedor
    WHERE ec.nombre = 'En tránsito'
)
SELECT 
    (SELECT total FROM ReservasActivas) AS reservas_activas,
    (SELECT ROUND(promedio_ocupacion, 2) FROM OcupacionBuques) AS ocupacion_promedio_buques_porcentaje,
    (SELECT total FROM IngresosProyectados) AS ingresos_proyectados_usd,
    (SELECT total FROM ContenedoresTransito) AS contenedores_en_transito,
    (
        SELECT COUNT(DISTINCT b.id_buque)
        FROM gestion_reserva.Buque b
        INNER JOIN gestion_reserva.EstadoEmbarcacion ee ON b.id_estado_embarcacion = ee.id_estado_embarcacion
        WHERE ee.nombre = 'Disponible'
    ) AS buques_disponibles,
    (
        SELECT COUNT(DISTINCT c.id_cliente)
        FROM gestion_reserva.Cliente c
        INNER JOIN gestion_reserva.Reserva r ON c.ruc = r.ruc_cliente
        WHERE r.fecha_registro >= CURRENT_DATE - INTERVAL '30 days'
    ) AS clientes_activos_mes;
```

**8. Reporte de Ocupación por Ruta**
```sql
-- Análisis de uso de rutas con estadísticas de reservas
SELECT 
    rt.codigo AS codigo_ruta,
    rt.origen,
    rt.destino,
    rt.duracion AS duracion_dias,
    rt.tarifa,
    COUNT(DISTINCT r.id_reserva) AS total_reservas,
    COUNT(DISTINCT CASE WHEN er.nombre = 'Confirmada' THEN r.id_reserva END) AS reservas_confirmadas,
    COUNT(DISTINCT CASE WHEN er.nombre = 'Activa' THEN r.id_reserva END) AS reservas_activas,
    COUNT(DISTINCT CASE WHEN er.nombre = 'Finalizada' THEN r.id_reserva END) AS reservas_finalizadas,
    COUNT(DISTINCT CASE WHEN er.nombre = 'Cancelada' THEN r.id_reserva END) AS reservas_canceladas,
    COALESCE(SUM(rc.cantidad), 0) AS total_contenedores_transportados,
    COALESCE(SUM(CASE WHEN er.nombre = 'Finalizada' THEN r.pago_total END), 0) AS ingresos_generados,
    MIN(r.fecha_registro) AS primera_reserva,
    MAX(r.fecha_registro) AS ultima_reserva,
    COUNT(DISTINCT r.ruc_cliente) AS clientes_diferentes
FROM gestion_reserva.Ruta rt
LEFT JOIN gestion_reserva.Reserva r ON rt.id_ruta = r.id_ruta
LEFT JOIN gestion_reserva.EstadoReserva er ON r.id_estado_reserva = er.id_estado_reserva
LEFT JOIN gestion_reserva.ReservaContenedor rc ON r.id_reserva = rc.id_reserva
WHERE r.fecha_registro >= CURRENT_DATE - INTERVAL '90 days'  -- Últimos 3 meses
GROUP BY rt.id_ruta, rt.codigo, rt.origen, rt.destino, rt.duracion, rt.tarifa
ORDER BY total_contenedores_transportados DESC;
```

**9. Creación de Nueva Reserva (Transacción Completa)**
```sql
-- Paso 1: Crear la reserva base
INSERT INTO gestion_reserva.Reserva (
    id_reserva,
    codigo,
    fecha_registro,
    id_estado_reserva,
    pago_total,
    ruc_cliente,
    id_agente_reservas,
    matricula_buque,
    id_ruta
)
VALUES (
    gen_random_uuid(),
    'RES-' || TO_CHAR(CURRENT_DATE, 'YYYYMMDD') || '-' || LPAD(nextval('seq_reserva')::TEXT, 3, '0'),
    CURRENT_DATE,
    (SELECT id_estado_reserva FROM gestion_reserva.EstadoReserva WHERE nombre = 'Confirmada'),
    NULL,  -- Se calculará después
    '20481234567',  -- RUC del cliente
    (SELECT id_agente_reservas FROM gestion_reserva.AgenteReservas LIMIT 1),
    'IMO-9347438',  -- Matrícula del buque
    (SELECT id_ruta FROM gestion_reserva.Ruta WHERE codigo = 'RUT-001')
)
RETURNING id_reserva;

-- Paso 2: Asignar contenedores a la reserva
INSERT INTO gestion_reserva.ReservaContenedor (
    id_reserva_contenedor,
    id_reserva,
    id_contenedor,
    fecha_asignacion,
    cantidad
)
SELECT 
    gen_random_uuid(),
    (SELECT id_reserva FROM gestion_reserva.Reserva WHERE codigo = 'RES-007'),
    id_contenedor,
    CURRENT_DATE,
    2  -- Cantidad de este tipo de contenedor
FROM gestion_reserva.Contenedor 
WHERE codigo IN ('CONT-001', 'CONT-003');  -- Contenedores seleccionados

-- Paso 3: Actualizar estado de contenedores a 'En tránsito'
UPDATE gestion_reserva.Contenedor 
SET id_estado_contenedor = (
    SELECT id_estado_contenedor 
    FROM gestion_reserva.EstadoContenedor 
    WHERE nombre = 'En tránsito'
)
WHERE codigo IN ('CONT-001', 'CONT-003');

-- Paso 4: Calcular y actualizar pago total
UPDATE gestion_reserva.Reserva r
SET pago_total = (
    SELECT 
        COALESCE(SUM(tc.costo * rc.cantidad), 0) + rt.tarifa
    FROM gestion_reserva.ReservaContenedor rc
    INNER JOIN gestion_reserva.Contenedor c ON rc.id_contenedor = c.id_contenedor
    INNER JOIN gestion_reserva.TipoContenedor tc ON c.id_tipo_contenedor = tc.id_tipo_contenedor
    CROSS JOIN gestion_reserva.Ruta rt
    WHERE rc.id_reserva = r.id_reserva
    AND rt.id_ruta = r.id_ruta
)
WHERE r.codigo = 'RES-007';

-- Paso 5: Vincular con operación marítima
INSERT INTO gestion_reserva.ReservaOperacionMaritima (
    id_reserva_operacion_maritima,
    id_reserva,
    id_operacion_maritima,
    fecha_vinculacion
)
VALUES (
    gen_random_uuid(),
    (SELECT id_reserva FROM gestion_reserva.Reserva WHERE codigo = 'RES-007'),
    (SELECT id_operacion_maritima FROM gestion_reserva.OperacionMaritima WHERE codigo = 'OM-2025-001'),
    CURRENT_DATE
);

-- Paso 6: Vincular con operación terrestre
INSERT INTO gestion_reserva.ReservaOperacionTerrestre (
    id_reserva_operacion_terrestre,
    id_reserva,
    id_operacion_terrestre,
    fecha_vinculacion
)
VALUES (
    gen_random_uuid(),
    (SELECT id_reserva FROM gestion_reserva.Reserva WHERE codigo = 'RES-007'),
    (SELECT id_operacion_terrestre FROM gestion_reserva.OperacionTerrestre WHERE codigo = 'OT-2025-001'),
    CURRENT_DATE
);
```

**10. Actualización de Estado de Reserva**
```sql
-- Cambiar estado de reserva
UPDATE gestion_reserva.Reserva 
SET id_estado_reserva = (
    SELECT id_estado_reserva 
    FROM gestion_reserva.EstadoReserva 
    WHERE nombre = 'Cancelada'  -- Valores: 'Confirmada', 'Activa', 'Finalizada', 'Cancelada'
)
WHERE codigo = 'RES-004';

-- Si se cancela, liberar contenedores asociados
UPDATE gestion_reserva.Contenedor 
SET id_estado_contenedor = (
    SELECT id_estado_contenedor 
    FROM gestion_reserva.EstadoContenedor 
    WHERE nombre = 'Disponible'
)
WHERE id_contenedor IN (
    SELECT rc.id_contenedor
    FROM gestion_reserva.ReservaContenedor rc
    INNER JOIN gestion_reserva.Reserva r ON rc.id_reserva = r.id_reserva
    WHERE r.codigo = 'RES-004'
);

-- Registrar la cancelación con un agente
UPDATE gestion_reserva.Reserva
SET id_estado_reserva = (
    SELECT id_estado_reserva 
    FROM gestion_reserva.EstadoReserva 
    WHERE nombre = 'Cancelada'
)
WHERE codigo = 'RES-004'
AND id_estado_reserva = (
    SELECT id_estado_reserva 
    FROM gestion_reserva.EstadoReserva 
    WHERE nombre IN ('Confirmada', 'Activa')
);
```
