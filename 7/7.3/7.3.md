# 7.3. Carga Masiva de Datos (Opcional)

## Introducción

El poblamiento de bases de datos con datos realistas es crucial para:
- **Pruebas de rendimiento**: Evaluar el comportamiento del sistema con volúmenes significativos de datos
- **Validación de restricciones**: Verificar que las reglas de integridad funcionen correctamente
- **Desarrollo y pruebas**: Simular escenarios reales sin exponer datos sensibles
- **Demostración**: Presentar el sistema con datos coherentes y representativos

Para este proyecto, optamos por automatizar la generación de datos utilizando **Python** y la librería **Faker**, lo que nos permite crear miles de registros consistentes en minutos.

## Herramientas Utilizadas

### Faker: Generación de Datos Falsos

[Faker](https://faker.readthedocs.io/) es una librería de Python que genera datos falsos pero realistas para múltiples propósitos:

**Características principales:**
- Generación de nombres, direcciones, correos electrónicos, teléfonos, etc.
- Soporte para múltiples idiomas y localizaciones
- Datos coherentes y con formato válido
- Altamente personalizable y extensible

**Instalación:**

```bash
pip install faker
```

**Uso básico:**
```python
from faker import Faker

# Configurar para español
fake = Faker('es_ES')

# Generar datos
nombre = fake.first_name()          # 'Carlos'
email = fake.email()                # 'maria@example.com'
direccion = fake.address()          # 'Calle Mayor 123, Madrid'
fecha = fake.date_between('-1y')    # '2024-03-15'
```

## Estrategia de Poblamiento

### 1. Generación de Archivos CSV

Cada tabla del sistema tiene un script Python dedicado que:
1. Define la estructura de datos según el esquema SQL
2. Genera datos falsos respetando las restricciones (CHECK, UNIQUE, NOT NULL)
3. Mantiene integridad referencial entre tablas relacionadas
4. Exporta los datos a archivos CSV numerados

### 2. Importación Masiva con PostgreSQL

Los archivos CSV se importan usando el comando `COPY` de PostgreSQL:
```sql
COPY esquema.Tabla(columna1, columna2, ...)
FROM 'ruta/archivo.csv'
WITH (FORMAT csv, HEADER true, DELIMITER ',', ENCODING 'UTF8');
```

**Ventajas de este enfoque:**

- Alta velocidad de inserción (miles de registros por segundo)
- Manejo eficiente de grandes volúmenes
- Separación entre generación y carga de datos
- Fácil inspección y modificación de datos antes de importar

## Ventajas del Enfoque Implementado

## Ejemplo Práctico

A continuación se muestra un ejemplo simplificado de cómo se genera una tabla de clientes:

```python
from faker import Faker
import csv
import uuid

fake = Faker('es_ES')

# Generar 50 clientes
clientes = []
for i in range(50):
    cliente = {
        'id_cliente': str(uuid.uuid4()),
        'ruc': f"{fake.random_number(digits=11)}",
        'razon_social': fake.company(),
        'direccion': fake.address().replace('\n', ', '),
        'email': fake.email()
    }
    clientes.append(cliente)

# Guardar en CSV
with open('clientes.csv', 'w', newline='', encoding='utf-8') as f:
    writer = csv.DictWriter(f, fieldnames=cliente.keys())
    writer.writeheader()
    writer.writerows(clientes)
```
