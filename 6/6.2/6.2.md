# 6.2. Módulo de Gestión del Personal y Tripulación

# Diseño Físico  
## 𝄜 Tabla: TRIPULANTE  

---

### **1. Justificación de la selección**

La tabla **TRIPULANTE** fue seleccionada para el análisis físico por las siguientes razones:

- **Volumen de datos:** Se espera que el número de tripulantes crezca constantemente en el tiempo debido a la expansión de operaciones marítimas y la contratación de nuevo personal.  
- **Criticidad:** Contiene información operativa clave del personal embarcado, necesaria para gestionar asignaciones, disponibilidad y contratos.  
- **Relevancia:** Está directamente relacionada con procesos transaccionales de programación de operaciones marítimas, asignación de roles y gestión de recursos humanos.  

Por estos motivos, se considera una **tabla de alta criticidad y movimiento** dentro del módulo de tripulación.

---

### **2. Estimación de cantidad de registros**

Se estiman los siguientes valores de registros para la tabla **TRIPULANTE**, considerando un crecimiento progresivo anual basado en los supuestos de expansión organizacional.

#### **Supuestos:**
- Año base: **1,500 tripulantes registrados.**
- Crecimiento anual:
  - Año 2 → **+7%**
  - Año 3 → **+8%**

#### **Proyección:**

| Año | Nuevos Tripulantes | Total Tripulantes | Crecimiento |
|-----|--------------------|------------------|--------------|
| 1 | 1,500 | 1,500 | --- |
| 2 | 1,605 | 3,105 | 7% |
| 3 | 1,674 | 4,779 | 8% |

---

### **3. Script SQL de creación de tabla**

```sql
CREATE TABLE MODULO_TRIPULACION.TRIPULANTE (
    id_tripulante INT PRIMARY KEY,
    id_empleado INT NOT NULL UNIQUE, -- Relación 1:1 con Empleado
    disponibilidad BOOLEAN NOT NULL DEFAULT TRUE,
    nacionalidad VARCHAR(50) NOT NULL,
    id_contrato INT NOT NULL, -- Cada tripulante tiene un contrato
    CONSTRAINT fk_tripulante_empleado FOREIGN KEY (id_empleado) 
        REFERENCES MODULO_TRIPULACION.EMPLEADO(id_empleado)
        ON DELETE RESTRICT
        ON UPDATE CASCADE,
    CONSTRAINT fk_tripulante_contrato FOREIGN KEY (id_contrato) 
        REFERENCES MODULO_TRIPULACION.CONTRATO(id_contrato)
        ON DELETE RESTRICT
        ON UPDATE CASCADE
);

```
### **4. Estructura del registro**

| Atributo | Formato | Tamaño |
|-----------|----------|--------|
| id_tripulante | INT | 4 bytes |
| id_empleado | INT | 4 bytes |
| disponibilidad | BOOLEAN | 1 byte |
| nacionalidad | VARCHAR(50) | 50 bytes |
| id_contrato | INT | 4 bytes |
| **Total estimado por registro:** |  | **63 bytes** |

---


### **5. Resumen de parámetros físicos**

| Parámetro | Valor |
|------------|--------|
| Tamaño de bloque | 8 KB |
| Tamaño de extent | --- |
| Estructura del registro lógico | 63 bytes |
| Overhead de fila | 23 bytes |
| Filas por bloque | 76 |
| Espacio libre reservado | `ALTER TABLE tripulante SET (FILLFACTOR = 80);` |
| **Tamaño total estimado (año 3)** | **≈ 504 KB** |

---


### **6. Cálculos**

**Espacio libre por bloque:**  
(8,192 − 24) × 0.8 = **6,534.4 bytes por bloque**

**Cantidad de filas por bloque:**  
6,534.4 ÷ (63 + 23) = 6,534.4 ÷ 86 ≈ **75.97 ≈ 76 filas por bloque**

**Cantidad de bloques necesarios:**  
4,779 ÷ 76 ≈ **62.88 ≈ 63 bloques**

**Tamaño total estimado:**  
63 × 8,192 = **516,096 bytes ≈ 504 KB**

## 𝄜 Tabla: BUQUE

---

### **1. Justificación de la selección**

La tabla **BUQUE** fue seleccionada para el análisis físico por las siguientes razones:

- **Volumen de datos:** Se espera un crecimiento sostenido del número de embarcaciones debido a la ampliación de rutas marítimas y adquisición de nuevos buques.  
- **Criticidad:** Almacena información fundamental para la planificación de operaciones, asignación de tripulaciones y control logístico.  
- **Relevancia:** Se relaciona directamente con operaciones marítimas, mantenimiento y seguimiento en tiempo real de las embarcaciones.  

Por ello, **BUQUE** es una tabla clave dentro del sistema de gestión operativa marítima.

---

### **2. Estimación de cantidad de registros**

Se estiman los siguientes valores de registros para la tabla **BUQUE**, considerando un crecimiento anual progresivo de **7% el primer año** y **8% el segundo**, a partir de un total de **400 buques en el año base**.

#### **Supuestos:**
- Año base: 400 buques registrados.  
- Crecimiento anual:  
  - Año 2 → +7%  
  - Año 3 → +8%  

#### **Proyección:**

| Año | Nuevos Buques | Total Buques | Crecimiento |
|-----|----------------|---------------|--------------|
| 1 | 400 | 400 | --- |
| 2 | 28 | 428 | 7% |
| 3 | 34 | 462 | 8% |

---

### **3. Script SQL de creación de tabla**

```sql
CREATE TABLE MODULO_TRIPULACION.BUQUE (
    matricula VARCHAR(20) PRIMARY KEY,
    nombre VARCHAR(100) NOT NULL,
    capacidad INT NOT NULL,
    id_estado_embarcacion INT NOT NULL,
    peso DECIMAL(15,2) NOT NULL,
    ubicacion_actual VARCHAR(100),
    CONSTRAINT fk_buque_estado FOREIGN KEY (id_estado_embarcacion)
        REFERENCES MODULO_TRIPULACION.ESTADO_EMBARCACION(id_estado_embarcacion)
        ON DELETE RESTRICT
        ON UPDATE CASCADE
);
```

### **4. Estructura del registro**

| Atributo | Formato | Tamaño |
|-----------|----------|--------|
| matricula | VARCHAR(20) | 20 bytes |
| nombre | VARCHAR(100) | 100 bytes |
| capacidad | INT | 4 bytes |
| id_estado_embarcacion | INT | 4 bytes |
| peso | DECIMAL(15,2) | 8 bytes |
| ubicacion_actual | VARCHAR(100) | 100 bytes |
| **Total estimado por registro:** |  | **236 bytes** |


### **5. Resumen de parámetros físicos**

| Parámetro | Valor |
|------------|--------|
| Tamaño de bloque | 8 KB |
| Tamaño de extent | --- |
| Estructura del registro lógico | 236 bytes |
| Overhead de fila | 23 bytes |
| Filas por bloque | 26 |
| Espacio libre reservado | `ALTER TABLE buque SET (FILLFACTOR = 80);` |
| **Tamaño total estimado (año 3)** | **≈ 144 KB** |

### **6. Cálculos**

**Espacio libre por bloque:**  
(8,192 − 24) × 0.8 = **6,534.4 bytes por bloque**

**Cantidad de filas por bloque:**  
6,534.4 ÷ (236 + 23) = 6,534.4 ÷ 259 ≈ **25.23 ≈ 26 filas por bloque**

**Cantidad de bloques necesarios:**  
462 ÷ 26 ≈ **17.76 ≈ 18 bloques**

**Tamaño total estimado:**  
18 × 8,192 = **147,456 bytes ≈ 144 KB**
