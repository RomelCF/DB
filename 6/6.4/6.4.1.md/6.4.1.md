# 6.4.1. Módulo de Gestión de Mantenimiento Logístico

## 𝄜 Tabla: OperacionMantenimiento

### **1. Justificación de la selección**

La tabla **OperacionMantenimiento** fue seleccionada para el análisis físico por las siguientes razones:

  * **Volumen de datos:** Se espera un crecimiento constante de operaciones de mantenimiento debido al aumento de
    activos y solicitudes correctivas.
  * **Criticidad:** Es el núcleo del proceso de mantenimiento, conectando solicitudes, planes y ejecución.
  * **Relevancia:** Relaciona múltiples entidades del sistema como activos, técnicos, repuestos y órdenes, siendo
    clave para trazabilidad y control.

Por ello, se considera una **tabla de alta criticidad y movimiento** dentro del módulo logístico.

-----

### **2. Estimación de cantidad de registros**

#### **Supuestos:**

  * Año base: **2,000 operaciones de mantenimiento registradas.**
  * Crecimiento anual:
      * Año 2 → **+10%**
      * Año 3 → **+12%**

#### **Proyección:**

| Año | Nuevas Operaciones | Total Operaciones | Crecimiento |
|:---|:---|:---|:---|
| 1 | 2,000 | 2,000 | --- |
| 2 | 200 | 2,200 | 10% |
| 3 | 264 | 2,464 | 12% |

-----

### **3. Script SQL de creación de tabla**

```sql
CREATE TABLE mantenimiento_logistico.OperacionMantenimiento (
    id_operacion_mantenimiento CHAR(36) PRIMARY KEY DEFAULT gen_random_uuid()::TEXT,
    id_operacion CHAR(36) NOT NULL UNIQUE,
    id_plan_mantenimiento CHAR(36),
    id_solicitud_mantenimiento CHAR(36),
    CONSTRAINT fk_op_mantenimiento_operacion FOREIGN KEY (id_operacion)
        REFERENCES mantenimiento_logistico.Operacion(id_operacion)
        ON DELETE RESTRICT
        ON UPDATE CASCADE,
    CONSTRAINT fk_op_mantenimiento_plan FOREIGN KEY (id_plan_mantenimiento)
        REFERENCES mantenimiento_logistico.PlanMantenimiento(id_plan_mantenimiento)
        ON DELETE RESTRICT
        ON UPDATE CASCADE,
    CONSTRAINT fk_op_mantenimiento_solicitud FOREIGN KEY (id_solicitud_mantenimiento)
        REFERENCES mantenimiento_logistico.SolicitudMantenimiento(id_solicitud_mantenimiento)
        ON DELETE RESTRICT
        ON UPDATE CASCADE
);
```

## 𝄜 Tabla: Repuesto

### **1. Justificación de la selección**

La tabla **Repuesto** fue seleccionada por las siguientes razones:

  * **Volumen de datos:** Se espera un crecimiento sostenido del catálogo de repuestos por incorporación de nuevos
    activos y tareas.
  * **Criticidad:** Es esencial para el control de inventario, costos y disponibilidad de piezas.
  * **Relevancia:** Se vincula directamente con operaciones de mantenimiento y consumo histórico.

Por ello, **Repuesto** es una tabla clave en la gestión de recursos logísticos.

-----

### **2. Estimación de cantidad de registros**

#### **Supuestos:**

  * Año base: **500 repuestos registrados.**
  * Crecimiento anual:
      * Año 2 → **+15%**
      * Año 3 → **+20%**

#### **Proyección:**

| Año | Nuevos Repuestos | Total Repuestos | Crecimiento |
|:---|:---|:---|:---|
| 1 | 500 | 500 | --- |
| 2 | 75 | 575 | 15% |
| 3 | 115 | 690 | 20% |

-----

### **3. Script SQL de creación de tabla**

```sql
CREATE TABLE mantenimiento_logistico.Repuesto (
    id_repuesto CHAR(36) PRIMARY KEY DEFAULT gen_random_uuid()::TEXT,
    codigo VARCHAR(20) NOT NULL UNIQUE,
    nombre VARCHAR(100) NOT NULL,
    stock INT NOT NULL DEFAULT 0,
    stock_minimo INT NOT NULL DEFAULT 0,
    CONSTRAINT chk_stock CHECK (stock >= 0),
    CONSTRAINT chk_stock_minimo CHECK (stock_minimo >= 0)
);
```
