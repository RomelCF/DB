# 6.3. Módulo / Integrante 3

## Tabla: OperacionMaritima

---

### **1. Justificación de la selección**

La tabla **OperacionMaritima** fue seleccionada para el análisis físico por las siguientes razones:

- **Volumen de datos:** Se espera un alto volumen de operaciones marítimas debido al incremento en el tráfico de contenedores y la expansión de rutas comerciales.
- **Criticidad:** Almacena información esencial para el seguimiento en tiempo real de operaciones marítimas, gestión de contenedores y monitoreo del progreso de cada travesía.
- **Relevancia:** Se relaciona directamente con la planificación logística, asignación de buques, control de estatus de navegación y optimización de rutas.

Por ello, **OperacionMaritima** es una tabla central dentro del sistema de gestión de operaciones logísticas y marítimas.

---

### **2. Estimación de cantidad de registros**

Se estiman los siguientes valores de registros para la tabla **OperacionMaritima**, considerando un crecimiento anual progresivo de **12% el primer año** y **15% el segundo**, a partir de un total de **8,500 operaciones en el año base**.

#### **Supuestos:**
- Año base: 8,500 operaciones marítimas registradas.
- Crecimiento anual:
  - Año 2 → +12%
  - Año 3 → +15%

#### **Proyección:**

| Año | Nuevas Operaciones | Total Operaciones | Crecimiento |
|-----|---------------------|-------------------|-------------|
| 1   | 8,500               | 8,500             | ---         |
| 2   | 1,020               | 9,520             | 12%         |
| 3   | 1,428               | 10,948            | 15%         |

---

### **3. Script SQL de creación de tabla**

```sql
CREATE TABLE OperacionMaritima (
    id_operacion_maritima INT PRIMARY KEY,
    id_operacion INT NOT NULL UNIQUE, -- Relacion 1:1 con Operacion
    codigo VARCHAR(20) NOT NULL UNIQUE, -- Codigo UK
    cantidad_contenedores INT NOT NULL,
    id_estatus_navegacion INT NOT NULL,
    porcentaje_trayecto DECIMAL(5,2) NOT NULL DEFAULT 0.00, -- Porcentaje del trayecto completado
    matricula VARCHAR(20) NOT NULL, -- Buque asignado
    CONSTRAINT fk_op_maritima_operacion FOREIGN KEY (id_operacion) 
        REFERENCES Operacion(id_operacion)
        ON DELETE RESTRICT
        ON UPDATE CASCADE,
    CONSTRAINT fk_op_maritima_estatus FOREIGN KEY (id_estatus_navegacion) 
        REFERENCES EstatusNavegacion(id_estatus_navegacion)
        ON DELETE RESTRICT
        ON UPDATE CASCADE,
    CONSTRAINT fk_op_maritima_buque FOREIGN KEY (matricula) 
        REFERENCES Buque(matricula)
        ON DELETE RESTRICT
        ON UPDATE CASCADE,
    CONSTRAINT chk_cantidad_contenedores CHECK (cantidad_contenedores >= 0), -- Cantidad mayor o igual a 0
    CONSTRAINT chk_porcentaje CHECK (porcentaje_trayecto BETWEEN 0 AND 100) -- Porcentaje entre 0 y 100
);
```

---

### **4. Estructura del registro**

| Atributo                  | Formato        | Tamaño   |
|---------------------------|----------------|----------|
| id_operacion_maritima     | INT            | 4 bytes  |
| id_operacion              | INT            | 4 bytes  |
| codigo                    | VARCHAR(20)    | 20 bytes |
| cantidad_contenedores     | INT            | 4 bytes  |
| id_estatus_navegacion     | INT            | 4 bytes  |
| porcentaje_trayecto       | DECIMAL(5,2)   | 4 bytes  |
| matricula                 | VARCHAR(20)    | 20 bytes |
| **Total estimado por registro:** |          | **60 bytes** |

---

### **5. Resumen de parámetros físicos**

| Parámetro                              | Valor                                                    |
|----------------------------------------|----------------------------------------------------------|
| Tamaño de bloque                       | 8 KB                                                     |
| Tamaño de extent                       | ---                                                      |
| Estructura del registro lógico         | 60 bytes                                                 |
| Overhead de fila                       | 23 bytes                                                 |
| Filas por bloque                       | 78                                                       |
| Espacio libre reservado                | `ALTER TABLE operacion_maritima SET (FILLFACTOR = 80);` |
| **Tamaño total estimado (año 3)**     | **≈ 1.09 MB**                                            |

---

### **6. Cálculos**

**Espacio útil por bloque:**  
(8,192 − 24) × 0.8 = **6,534.4 bytes por bloque**

**Cantidad de filas por bloque:**  
6,534.4 ÷ (60 + 23) = 6,534.4 ÷ 83 ≈ **78.72 ≈ 78 filas por bloque**

**Cantidad de bloques necesarios:**  
10,948 ÷ 78 ≈ **140.36 ≈ 141 bloques**

**Tamaño total estimado:**  
141 × 8,192 = **1,155,072 bytes ≈ 1.09 MB**

## Tabla: RutaMaritima

---

### **1. Justificación de la selección**

La tabla **RutaMaritima** fue seleccionada para el análisis físico por las siguientes razones:

- **Volumen de datos:** Se espera un crecimiento moderado pero constante en el número de rutas marítimas debido a la apertura de nuevas conexiones comerciales entre puertos y la optimización de rutas existentes.
- **Criticidad:** Almacena información esencial para la planificación de trayectos, cálculo de tiempos de navegación, estimación de costos operativos y asignación eficiente de buques.
- **Relevancia:** Se relaciona directamente con la optimización logística, definición de rutas comerciales estratégicas y gestión de distancias entre puertos de origen y destino.

Por ello, **RutaMaritima** es una tabla fundamental dentro del sistema de gestión de operaciones y planificación de navegación.

---

### **2. Estimación de cantidad de registros**

Se estiman los siguientes valores de registros para la tabla **RutaMaritima**, considerando un crecimiento anual progresivo de **8% el primer año** y **10% el segundo**, a partir de un total de **1,200 rutas en el año base**.

#### **Supuestos:**
- Año base: 1,200 rutas marítimas registradas.
- Crecimiento anual:
  - Año 2 → +8%
  - Año 3 → +10%

---

### **3. Script SQL de creación de tabla**

```sql
CREATE TABLE RutaMaritima (
    id_ruta_maritima INT PRIMARY KEY,
    id_ruta INT NOT NULL UNIQUE, -- Relacion 1:1 con Ruta
    codigo VARCHAR(20) NOT NULL UNIQUE, -- Codigo UK
    id_puerto_origen INT NOT NULL,
    id_puerto_destino INT NOT NULL,
    distancia INT NOT NULL, -- Distancia en millas nauticas
    CONSTRAINT fk_ruta_maritima_ruta FOREIGN KEY (id_ruta) 
        REFERENCES Ruta(id_ruta)
        ON DELETE RESTRICT
        ON UPDATE CASCADE,
    CONSTRAINT fk_ruta_puerto_origen FOREIGN KEY (id_puerto_origen) 
        REFERENCES Puerto(id_puerto)
        ON DELETE RESTRICT
        ON UPDATE CASCADE,
    CONSTRAINT fk_ruta_puerto_destino FOREIGN KEY (id_puerto_destino) 
        REFERENCES Puerto(id_puerto)
        ON DELETE RESTRICT
        ON UPDATE CASCADE,
    CONSTRAINT chk_distancia CHECK (distancia > 0), -- Distancia mayor a 0
    CONSTRAINT chk_puertos_diferentes CHECK (id_puerto_origen != id_puerto_destino) -- Puertos diferentes
);
```
